# Q:在神经网络中引入残差项 f(x) = x + g(x)得到 ResNet。为什么这样的操作要叫做“残差”？

在 ResNet 中，把 $f(x) = x + g(x)$ 里的 $g(x)$ 称为“残差”，这个名字来自统计学和回归分析中的**残差**概念。

### 1. 统计学中的“残差”
在回归问题里，假设我们想用模型去逼近一个真实函数 $H(x)$。如果直接让模型去学 $H(x)$，就是在学习完整的映射。  
而“残差” = **观测值 $-$ 预测值**，代表当前模型还没有解释掉的部分。

### 2. ResNet 里的“残差”
ResNet 的设计思想是把整个模块希望学到的底层映射记作 $H(x)$，但它不直接让权重层去拟合 $H(x)$，而是让它们去拟合**另一个量**：
$$g(x) = H(x) - x$$
这里的 $g(x)$ 就是输入 $x$ 与目标输出 $H(x)$ 之间的**差**，这正是“残差”的含义。  
然后将这个残差加回输入，得到最终输出：
$$f(x) = x + g(x) = H(x)$$
所以，**$g(x)$ 逼近的就是残差**，网络实际在学习“输出和输入相比还需要补充多少”。

### 3. 为什么这样命名很贴切
- **学习目标的转变**：原来网络要直接学习 $H(x)$，现在变成了学习 $H(x)-x$。如果最优映射接近恒等映射（$H(x) \approx x$），那残差就接近 0。让网络把残差推向 0 比直接拟合恒等映射要容易得多。
- **优化角度**：引入“残差学习”后，深层网络可以更容易地让梯度通过“捷径” $x$ 回传，解决了退化问题，同时名字也直观反映了该模块在学习“残差”。

因此，叫“残差网络”就是因为**网络的核心是显式地学习输入与目标输出之间的残差**，而不是完整的输出。

# Q:为什么引入残差会使网络的加深，至少不会使模型的结果变得更差？

这背后的核心原因，可以总结为一句话：**残差结构让网络“更容易”学会什么都不做（恒等映射），从而避免加深带来的副作用。**

当网络加深而效果变差（退化问题），本质上不是过拟合，而是**深层网络连训练误差都降不下去**。这说明优化变得异常困难。

引入残差后，情况就不同了。

### 1. 把“难以企及的目标”变成“易于归零的修正”
假设我们想学到一个理想的映射 $H(x)$。

-   **普通网络**：需要直接从 $x$ 拟合 $H(x)$。如果深层的最优解接近恒等映射（即 $H(x) \approx x$），这就要求权重层去精确拟合一个复杂的恒等函数，这对非线性层来说非常困难。
-   **残差网络**：构建为 $H(x) = F(x) + x$，让权重层去学残差 $F(x) = H(x) - x$。
    -   如果恒等映射已经足够好，网络只需要把 $F(x)$ 的所有权重推向 0，输出就近似等于 $x$。
    -   **把权重推向 0，比让权重通过非线性去模拟恒等映射，要简单得多。** 这是残差网络的核心洞见。

### 2. 提供了一条“高速梯度公路”
在反向传播时，残差块的输出对输入的梯度为：
$$\frac{\partial (x + F(x))}{\partial x} = 1 + \frac{\partial F(x)}{\partial x}$$
那个常数 **1** 至关重要。它保证了梯度可以从深层直接、无损地传到浅层。即使 $\frac{\partial F(x)}{\partial x}$ 非常小，梯度也至少为 1，有效缓解了梯度消失。

这意味着：
即使层数极深，网络也总能得到一个稳定的梯度流来驱动训练，优化不会因层数变深而崩溃。这是“不会变得更差”的优化基础。

### 3. 可以看作“浅层网络的集成”
从另一个角度看，一个有 $n$ 个残差块的网络，数据流动的路径数量是 $2^n$ 量级，相当于**众多不同深度网络的集成**。

当去掉某些层时，相当于激活了网络中一些更短的子路径，整体性能不会断崖式下跌。反过来，即便新增的层没学到任何有用信息（退化为恒等），它们也只是传递了来自短路径的信息，不会破坏已有的好结果。

### 总结
**因为残差学习把问题从“学习一个复杂的恒等映射”简化为“把残差推向零”，并同时提供了保证梯度的恒等路径。** 所以增加层数后，网络在最坏情况下可以自动退化为较浅的网络，从而确保加深后的模型性能下限至少不低于较浅的模型。
