# Q:如果样本不平衡，使用softmax回归会有什么问题？
当训练样本类别严重不平衡时，直接使用 Softmax 回归（即多分类逻辑回归）会引发以下几个关键问题：

1. **模型偏向多数类**  
   Softmax 回归通过最小化交叉熵损失来学习决策边界。由于多数类样本占主导，模型会优先降低多数类的预测误差，导致决策边界严重偏向少数类。最终模型倾向于将所有样本预测为多数类，少数类的召回率极低。

2. **少数类梯度被淹没**  
   在梯度下降过程中，多数类产生的梯度信号远强于少数类。少数类样本对参数更新的贡献微弱，模型难以有效学习其特征。尤其在深度网络等复杂模型中，少数类梯度可能完全被噪声掩盖。

3. **概率校准失真**  
   Softmax 输出的概率本质上是特征线性组合的指数归一化结果。在不平衡数据上，即使模型对多数类输出高概率（如 0.9），对少数类输出低概率（如 0.1），这种概率分布也往往反映的是训练集的先验偏差，而非真实的类别置信度。实际应用时，需通过 Platt scaling 或 isotonic regression 重新校准。

4. **评估指标误导**  
   若直接用准确率评估，在不平衡比例如 99:1 时，模型即使全猜多数类也能达到 99% 准确率，但少数类完全未被识别。需改用精确率、召回率、F1、AUC-ROC 或 PR 曲线。

5. **过拟合风险增加**  
   少数类样本数量极少，模型容易“记住”这些样本而非学习泛化特征。尤其在特征维数高时，线性决策边界可能恰好穿过几个少数类样本，导致测试集上少数类性能崩溃。

### 缓解方法
- **重采样**：对多数类降采样（如随机欠采样、NearMiss）或对少数类过采样（如 SMOTE、ADASYN）。
- **代价敏感学习**：为少数类分配更高的误分类惩罚权重，修改损失函数为加权交叉熵。
- **改变阈值**：训练后调整决策阈值，不再使用 0.5 或均匀先验，而是根据验证集的 F1 或 PR 曲线选择最优阈值。
- **使用其他算法**：集成方法（如 BalancedRandomForest、EasyEnsemble）、单类分类器（One-Class SVM）或基于异常检测的框架。
- **数据生成**：使用生成对抗网络（GAN）或变分自编码器（VAE）合成少数类样本。

在实际应用中，建议先明确不平衡产生的原因（天然分布偏差还是采样偏差），再结合业务需求（如医疗诊断更关注少数正例）选择合适策略。
